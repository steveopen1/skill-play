# API 安全渗透测试使用示例

> **参考说明**：本文档展示 Agent 驱动的 API 安全测试**可能的执行方式**
> - 实际测试流程应根据目标特性**灵活调整**
> - 测试顺序、策略选择、payload 构造均需**动态决策**
> - 本文档仅供参考，非固定测试流程

---

## 场景 2: 特定漏洞检查

### 用户请求
```
帮我检查有没有 SQL 注入漏洞
```

### Agent 执行流程

1. 识别所有带参数的端点
2. 针对每个参数测试 SQL 注入
3. 使用多种注入技术:
   - Boolean-based
   - Union-based
   - Error-based
   - Time-based Blind
4. 验证漏洞有效性
5. 输出具体位置和利用方式

---

## 场景 3: SPA 应用测试

### 用户请求
```
测试这个 Vue.js 应用的 API 安全
```

### Agent 执行流程

1. **SPA 检测**
   - 识别 SPA fallback 行为
   - 确定后端 API 不在主域名

2. **JS 分析**
   - 提取所有 JS 文件
   - 从 JS 中提取 API 路径
   - 发现内网地址或测试环境地址

3. **策略调整**
   - 切换到 SPA 深度分析策略
   - 针对提取到的端点测试

---

## 场景 4: JWT 认证测试

### 用户请求
```
检查这个 API 的 JWT 认证是否存在漏洞
```

### Agent 执行流程

1. 识别 JWT 使用
2. 测试以下漏洞:
   - `none` 算法
   - 弱密钥
   - 密钥混淆攻击
   - Token 过期时间
3. 检查认证绕过可能性

---

## 场景 5: 生成报告

### 用户请求
```
生成一份 API 安全测试报告
```

### 输出格式

```markdown
# API 安全渗透测试报告

## 执行摘要
- **目标**: https://api.example.com
- **扫描时间**: 2026-03-31 10:00 - 11:30
- **总请求数**: 1500
- **发现漏洞**: 8

## 技术栈
- 后端: Spring Boot
- 前端: Vue.js + Webpack
- 数据库: PostgreSQL
- WAF: 阿里云

## 漏洞详情

### 1. SQL Injection (Critical)
- **端点**: `/api/users?id=1`
- **修复**: 使用参数化查询

### 2. IDOR (High)
- **端点**: `/api/users/{id}/profile`
- **修复**: 实施访问控制检查
```
